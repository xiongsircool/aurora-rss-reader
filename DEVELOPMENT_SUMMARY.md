## 🏗️ 技术架构

### 后端 (Python FastAPI)
```
backend/
├── app/
│   ├── api/routes/
│   │   ├── feeds.py      # 订阅管理 (CRUD + 刷新)
│   │   ├── entries.py    # 文章管理
│   │   ├── ai.py         # AI 摘要和翻译
│   │   ├── opml.py       # OPML 导入导出
│   │   └── health.py     # 健康检查
│   ├── services/
│   │   ├── fetcher.py    # RSS 抓取和解析
│   │   └── ai.py         # GLM 客户端
│   ├── db/
│   │   ├── models.py     # SQLModel 数据模型
│   │   └── session.py    # 数据库会话
│   └── schemas/          # Pydantic 验证模型
└── scripts/
    ├── serve.py          # 开发服务器
    └── migrate.py        # 数据库迁移
```

## 📦 数据模型

### 数据库表
1. **feeds** - 订阅源
   - 基本信息、URL、分组、更新间隔
   - 最后检查时间、错误信息

2. **entries** - 文章条目
   - 标题、内容、摘要
   - 已读/收藏状态
   - Readability 清洗内容

3. **translations** - 翻译结果
   - 多语言翻译缓存
   - 标题、摘要、内容翻译

4. **summaries** - AI 摘要
   - 摘要文本
   - 语言标识

5. **fetch_logs** - 抓取日志
   - 状态、耗时、条目数
   - 错误信息

## 🔌 API 端点

### 订阅管理
- `GET /api/feeds` - 获取订阅列表
- `POST /api/feeds` - 添加订阅
- `PATCH /api/feeds/{id}` - 更新订阅
- `DELETE /api/feeds/{id}` - 删除订阅
- `POST /api/feeds/{id}/refresh` - 刷新订阅

### 文章管理
- `GET /api/entries` - 获取文章列表
- `PATCH /api/entries/{id}` - 更新文章状态

### AI 功能
- `POST /api/ai/summary` - 生成摘要
- `POST /api/ai/translate` - 翻译文章

### OPML
- `GET /api/opml/export` - 导出 OPML
- `POST /api/opml/import` - 导入 OPML

## 🚀 启动指南

### 快速启动

```bash
# 使用快速启动脚本
./start.sh

# 或手动启动
cd rss-desktop
pnpm dev
```

### 首次设置

1. 配置 API Key
   - 点击左上角设置图标 (⚙️)
   - 输入 GLM API Key
   - 保存设置

2. 添加订阅
   - 在左侧输入 RSS URL
   - 点击"添加订阅"

3. 开始使用
   - 浏览文章
   - 生成 AI 摘要
   - 翻译文章
