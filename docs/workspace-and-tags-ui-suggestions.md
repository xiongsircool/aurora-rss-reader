# 功能工作台 & 智能标签 · UI/逻辑改进建议

## 一、功能工作台（Workspace）

### 1. 左侧导航 (WorkspaceNav)
- **现状**：三个模块平铺，选中态只有背景色，折叠后仅剩图标，识别度一般。
- **建议**：
  - 选中项增加**左侧竖条**（accent 色），或小圆点，强化「当前所在」。
  - 折叠时用 **tooltip** 显示完整名称，避免悬停才知是什么。
  - 可选：为每个模块加一句**短副标题**（如「整理与收藏」「AI 标签」「全文检索」），帮助新用户理解差异。

### 2. 顶栏 (WorkspaceHeader)
- **现状**：Logo + 标题 + 主题 + 返回，信息清晰但略平。
- **建议**：
  - 在标题旁或下方加**当前模块名**（如「合集」「智能标签」「搜索」），或简单面包屑 `工作台 / 智能标签`，减少「不知道自己在哪」的感觉。
  - 返回按钮可带轻微 hover 动效（如箭头位移），增强可点击感。

### 3. 三栏布局与层次
- **现状**：三栏等权，视觉上容易「平」。
- **建议**：
  - **中间列表**：保持为主内容区。
  - **右侧详情**：用 `--bg-elevated` 或极轻的左侧阴影/边框，让「详情」在层级上略高于列表，主从更清晰。
  - 列表与详情之间的**分割线**可沿用 `--border-color`，保证与主题一致。

### 4. 空状态
- **现状**：无模块时只显示 `common.noData` 一段字。
- **建议**：
  - 统一成**空状态组件**：图标/插画 + 标题 + 一句引导（如「在左侧选择合集、标签或搜索」），风格与合集/标签的 empty state 一致。

## 二、智能标签（Tags）

### 1. 信息架构（逻辑）
- **现状**：左侧栏同时堆了「待分析统计 + 一键分析」「配置面板（订阅范围、区间重跑）」「我的标签」「视图（待分析/未打标签）」，首屏信息过密。
- **建议**：
  - **默认收起「配置」**：订阅范围、区间重跑放进「设置」齿轮下的折叠区或二级面板，默认只展示「待分析数量 + 分析按钮」和「我的标签 + 视图」。
  - **视图区分**：待分析 vs 未打标签 在界面上更明显——例如不同图标、或一行说明（「待分析：尚未打标签的新文章」「未打标签：历史中从未被打过标签的文章」），减少混淆。

### 2. 标签列表视觉
- **现状**：每个标签是「色点 + 名称 + 数字」，hover 才出编辑/删除，颜色多时略花。
- **建议**：
  - 色块改为**左侧细条**（2–3px 竖条或小圆点），或淡色底 + 圆角，减少抢眼。
  - 数字用**更小字重/次要色**，不喧宾夺主。
  - 编辑/删除可改为**常显图标**但低对比，hover 再高亮，或右键菜单，减少误触。

### 3. 区间重跑弹窗
- **现状**：时间、模式、订阅源、条数、说明文字全在一个长表单里，认知负担大。
- **建议**：
  - **分步**：Step 1 选时间范围 → Step 2 选订阅源（可选）→ Step 3 确认并执行；或
  - **折叠**：主界面只保留「时间范围 + 开始重跑」，把「模式、订阅源、条数」收进「高级选项」。
  - 执行结果（total/success/tagged/untagged）用**小结卡片**或表格展示，而不是长句。

### 4. 条目列表 (TagEntryList)
- **现状**：与 CollectionDetail 的列表在间距、圆角、选中态上略有差异，且「待分析」的 checkbox 与标题挤在一起。
- **建议**：
  - **统一列表规范**：卡片间距、padding、hover/选中态与合集、搜索一致（如统一用 `rounded-lg`、同一套 `--bg-hover` / accent 淡底）。
  - Checkbox 与标题之间留出固定间距，并保证**垂直对齐**（如 flex + items-start，checkbox 与标题首行对齐）。

### 5. 逻辑与反馈
- **分析中**：除按钮 loading 外，可在列表区域加**轻量进度提示**（如「正在为 3 篇文章打标签…」），减少用户重复点击。
- **无标签时**：若用户尚未创建任何标签，在点击「待分析」或「分析全部」时，可**先引导创建标签**（简短提示 + 跳转或弹窗），再进行分析。

## 三、可优先落地的改动（已实现部分）

1. **WorkspaceNav**：选中态左侧竖条 + 折叠时 tooltip。
2. **TagList**：配置区默认折叠；标签项改为左侧色条 + 数字弱化；统计卡片与按钮区域略收紧留白。
3. **TagEntryList**：列表项与 CollectionDetail 统一的圆角/间距/选中态；checkbox 对齐与间距。

以上建议可按优先级分阶段做，先做信息架构与层次（收配置、统一列表），再做视觉细节（色条、数字、空状态）。
