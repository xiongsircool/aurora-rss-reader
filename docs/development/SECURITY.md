# 🔒 Aurora RSS Reader 安全政策

## 📋 概述

Aurora RSS Reader 非常重视安全性。本安全政策说明如何负责任地披露安全漏洞，以及我们如何处理安全问题。

## 🎯 安全范围

### ✅ **安全考虑范围**
- **应用程序安全**：前端 XSS、CSRF、注入攻击等
- **API 安全**：后端认证、授权、数据验证等
- **数据安全**：敏感信息存储、传输加密等
- **依赖安全**：第三方库漏洞、供应链安全等
- **隐私保护**：用户数据处理、个人信息保护等

### 🔍 **重点关注的漏洞类型**
- 🔥 **高危**：
  - 远程代码执行 (RCE)
  - SQL 注入
  - 身份认证绕过
  - 敏感数据泄露
  - 跨站脚本 (XSS)

- ⚠️ **中危**：
  - 跨站请求伪造 (CSRF)
  - 权限提升
  - 信息泄露
  - 拒绝服务攻击 (DoS)

- 💡 **低危**：
  - 安全配置错误
  - 弱密码策略
  - 日志信息泄露
  - 版本信息泄露

## 🛡️ 安全措施

### 🔐 **我们采取的安全措施**

#### 代码安全
- ✅ **代码审查**：所有代码变更都经过安全审查
- ✅ **静态分析**：使用工具扫描潜在安全问题
- ✅ **依赖扫描**：定期检查第三方库漏洞
- ✅ **安全测试**：包含安全测试的开发流程

#### 数据保护
- ✅ **本地存储**：用户数据主要存储在本地
- ✅ **API 密钥**：敏感信息不会硬编码或提交
- ✅ **传输安全**：使用 HTTPS 加密通信
- ✅ **数据清理**：定期清理敏感日志和缓存

#### 运行时安全
- ✅ **权限控制**：最小权限原则
- ✅ **沙箱隔离**：Electron 安全最佳实践
- ✅ **更新机制**：及时更新依赖库
- ✅ **监控告警**：异常行为监控

### 🔍 **用户安全建议**

#### 基本安全措施
- 🔑 **API 密钥安全**：
  - 使用强密码
  - 定期更换 API 密钥
  - 不要在公共设备保存密钥

- 🌐 **网络安全**：
  - 确保在安全的网络环境中使用
  - 避免使用不信任的 WiFi 网络
  - 及时更新应用版本

- 📁 **数据安全**：
  - 定期备份重要数据
  - 注意保护本地数据库文件
  - 谨慎分享配置文件

## 🔒 隐私保护

### 📊 **我们收集的数据**
- **使用数据**：应用使用统计（本地存储）
- **错误日志**：应用崩溃和错误信息
- **配置信息**：用户设置和偏好
- **诊断数据**：性能和兼容性数据

### 🛡️ **我们不收集的数据**
- ❌ 个人身份信息
- ❌ RSS 内容内容
- ❌ 用户浏览历史
- ❌ API 密钥或凭证

### 🎯 **隐私原则**
- **最小化收集**：只收集必要的数据
- **本地存储**：数据主要存储在用户设备
- **透明处理**：明确告知数据处理方式
- **用户控制**：用户控制数据的使用和删除
