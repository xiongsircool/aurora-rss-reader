# 🌈 Aurora RSS Reader 行为准则

## 📖 前言

欢迎来到 Aurora RSS Reader 社区！我们致力于创建一个友好、包容和充满激情的社区。就像极光一样，我们相信每个成员都能为这个项目带来独特的光芒。

本行为准则适用于所有参与 Aurora RSS Reader 项目的人员，包括但不限于：
- 项目维护者和贡献者
- 在 GitHub 上的互动者
- 社区活动的参与者
- 使用 Aurora RSS Reader 的用户

## 🌟 我们的承诺

为了营造一个开放和友好的环境，我们承诺：

### 🤝 **尊重与包容**
- 尊重不同的观点和经验
- 欢迎各种背景的贡献者
- 接受建设性的批评和建议
- 关注对社区最有利的事情

### 💡 **积极沟通**
- 使用友善和包容的语言
- 耐心解答新手问题
- 分享知识和经验
- 保持专业和礼貌

### 🎨 **创造环境**
- 让每个成员都感到受欢迎
- 鼓励创新和实验
- 庆祝每一个成功的贡献
- 从失败中学习和成长

## ✅ 期望行为

### 🗣️ **在社区互动中**
- 🌈 **友善沟通**：使用积极、鼓励性的语言
- 🎯 **专注问题**：讨论技术和内容，而非个人
- 🤝 **相互尊重**：承认并感谢他人的贡献
- 🌍 **文化敏感**：尊重不同文化背景的表达方式

### 💻 **在代码贡献中**
- 📝 **清晰文档**：提供清晰的提交信息和文档
- 🧪 **充分测试**：确保代码质量
- 🔄 **建设性反馈**：提供具体、可操作的改进建议
- 📚 **知识分享**：在评审中分享最佳实践

### 🎯 **在问题讨论中**
- 🔍 **先搜索**：避免重复提问
- 📋 **详细信息**：提供完整的上下文和复现步骤
- 🙏 **表达感谢**：感谢帮助你的社区成员
- 🌱 **持续参与**：跟进你创建的议题

## 🚫 不当行为

### ❌ **绝对禁止**
- 🚫 **人身攻击**：侮辱、贬低、威胁或其他攻击性言论
- 🚫 **歧视行为**：基于种族、性别、年龄、宗教、性取向等的歧视
- 🚫 **骚扰行为**：不受欢迎的关注、恐吓或威胁
- 🚫 **隐私侵犯**：发布他人私人信息
- 🚫 **恶意行为**：故意破坏、散布错误信息或恶意代码

### ⚠️ **需要避免**
- 🔄 **垃圾信息**：无关的广告或重复内容
- 🔗 **恶意链接**：钓鱼网站、恶意软件链接
- 💻 **代码抄袭**：未经适当归因使用他人代码
- 🎭 **冒充行为**：冒充他人或虚假身份
- 📵 **商业推销**：未经允许的商业推广

## 🛠️ 执行和报告

### 📢 **如何报告不当行为**

如果你遇到或观察到不当行为，可以通过以下方式报告：

1. **GitHub 报告**：
   - 私信给项目维护者
   - 使用 GitHub 的举报功能
   - 发送邮件到：1666526339@qq.com

2. **紧急情况**：
   - 涉及人身安全的紧急威胁，请立即联系当地执法部门
   - 然后通知项目维护者采取必要措施

### ⚖️ **执行流程**

#### 第一步：初步评估
- 维护者会在 48 小时内响应报告
- 评估情况的严重性
- 决定采取适当措施

#### 第二步：调查过程
- 收集相关信息和证据
- 与相关人员沟通
- 保护报告者隐私

#### 第三步：采取措施
可能的行动包括：
- 📝 **私人警告**：教育引导，要求道歉
- 🔇 **暂时禁言**：限制在项目中的发言权限
- 🚫 **暂时封禁**：禁止参与项目一段时间
- ❌ **永久封禁**：完全禁止访问项目资源

#### 第四步：后续跟进
- 监控情况改善
- 评估措施效果
- 必要时调整策略

- 💡 **自嘲精神**：可以适度自嘲，但不要贬低他人

## 📚 具体场景指导

### 🔄 **代码审查中**
✅ **推荐做法**：
- "这个想法不错，我们可以考虑..."
- "我注意到这里可能有个边界情况..."
- "你有没有考虑过这种替代方案？"

❌ **避免说法**：
- "这样写太垃圾了"
- "明显你没懂需求"
- "这么简单都做错"

### 💬 **讨论分歧时**
✅ **推荐做法**：
- "我理解你的观点，让我解释我的想法..."
- "我们可能需要更多信息来决定..."
- "让我们找个对大家都好的方案"

❌ **避免说法**：
- "你完全错了"
- "我的想法才是最好的"
- "没必要讨论了"

### 🆘 **帮助新手时**
✅ **推荐做法**：
- "欢迎！很高兴看到你参与项目"
- "这里有一些资源可能对你有帮助"
- "每个人都是从小白开始的，别担心"

❌ **避免说法**：
- "这么简单的问题还要问？"
- "你先去学习一下再来"
- "RTFM (Read The Fucking Manual)"

## 🌈 特殊考量

### 🌍 **国际化社区**
- 📝 **语言支持**：支持多种语言交流
- ⏰ **时区差异**：耐心等待不同时区的回应
- 🎨 **文化习惯**：尊重不同的表达方式
- 📅 **节日差异**：理解不同的文化背景

### ♿ **无障碍参与**
- 🎯 **清晰表达**：使用清晰简洁的语言

- 🔤 **工具适配**：考虑辅助技术用户的需求
- ⚖️ **机会平等**：为所有人提供平等参与机会
